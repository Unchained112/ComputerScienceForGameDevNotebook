## 用户需求

基于 DEVELOPMENT_PLAN.md 继续推进批次4（算法·进阶与图算法，5页）和批次5（题型·数组/链表/树图，3页），共8个双语内容页。

## 产品概述

为 CS Notebook 站点创建8个 `/topics/<slug>` 专属内容页，每页包含双语正文（`<Bilingual>` 包装）、`.glass` 卡片分区、代码示例与复杂度标注。批次4的5页各含一个 SVG 交互可视化（play/step/reset 控件 + frame 逐步推进），批次5的3页以题解+代码为主、酌情加轻量交互。

## 核心功能

- **批次4**：dp（DP填表动画）、sliding-window（窗口滑动高亮）、two-pointers（双指针/快慢指针动画）、bfs-dfs（迷宫BFS+树DFS可视化）、shortest-path（Dijkstra松弛+A*对比）
- **批次5**：p-array-string（Two Sum/盛水容器/最长回文子串）、p-linked-list（翻转链表/环入口/合并有序链表）、p-tree-graph（层序遍历/右视图/拓扑排序）
- 每页 slug 与 outline.ts 一致，静态路由覆盖占位页
- 交付后回写进度追踪表

## Tech Stack

- **Astro + TypeScript + Tailwind CSS**（静态双语站点，现有项目技术栈）
- 交互可视化：Astro `<script>` + 原生 TypeScript + SVG DOM 操作
- 无新增依赖，完全复用现有基础设施

## Implementation Approach

每页在 `src/pages/topics/` 下创建 `<slug>.astro` 静态文件，自动覆盖 `[topic].astro` 动态占位。页面结构沿用 `sorting.astro` 版式：`<BaseLayout>` → `<article>` → `header`（小标签+标题+导语）+ `.glass` 卡片分区。交互可视化沿用批次3的 `greedy.astro`/`backtracking.astro` 模式：预生成 frames 数组 + `advance()` 逐步推进 + `langText()` 辅助函数 + `langchange` 事件监听。

**批次4交互可视化设计**：

- **dp**：LCS 填表动画，逐步高亮当前格 + 依赖格（上/左/对角线），最终回溯路径
- **sliding-window**：字符串数组条形图，左右指针 `lo`/`hi` 滑动，窗口内元素高亮，重复时左指针收缩
- **two-pointers**：有序数组两端向中间收拢找两数之和，命中/跳过状态着色
- **bfs-dfs**：小迷宫网格，BFS 层序扩展（队列波纹）vs DFS 深入（栈路径），可切换模式
- **shortest-path**：加权图节点+边，Dijkstra 逐步松弛高亮（已确定/松弛中/未访问），附 A* 说明

**批次5题解页设计**：每页3道经典题，每题含题意简述 + 思路 + C++ 代码 + 复杂度，用 `.glass` 卡片分隔。`p-array-string` 酌情加 Two Sum 哈希查找的轻量交互。

**性能**：所有交互在客户端 `<script>` 内运行，预计算 frames 数组后仅做 SVG DOM 渲染，单帧 O(n) 以下，无性能瓶颈。构建为静态站，无 SSR 开销。

## Implementation Notes

- 样式类严格复用：`glass`、`text-primary-cyan`、`text-ink-muted`、`text-func-amber`/`text-func-red`/`text-func-green`/`text-primary-purple`，不新建 CSS
- 代码块用 `<pre><code>` 展示，HTML 实体转义（`&lt;`/`&gt;`/`&#123;`/`&amp;`），与 sorting.astro 一致
- SVG viewBox 尺寸与现有页面一致（800x300~400），`class="w-full h-80 viz-stage"`
- 按钮控件统一 `data-ui` + `data-lang` 属性用于 langchange 同步
- 每个页面 ID 前缀唯一（如 `dp-`/`sw-`/`tp-`/`bd-`/`sp-`/`pa-`/`pl-`/`pt-`），避免 DOM ID 冲突
- 交付标准：`npm run build` 无报错，lint 无错误，进度追踪表回写

## Architecture Design

现有架构不变，仅新增8个静态页面文件：

- `BaseLayout` → 页面 `<article>` → `header` + `.glass` 卡片 + `<script>` 交互
- 双语切换由全局 `html[data-lang]` + `Bilingual` 组件 + `langchange` 事件驱动
- 导航高亮由 `SidebarItem` 基于 `Astro.url.pathname` 自动处理，无需改动
